Software requirements specification

Telegram channel collector

A private two-user web application for collecting, de-duplicating, and exporting Telegram channel links in preparation for large-scale token scraping.

Version 1.0 — Phase 1 Users 2 (private, login-gated) Target 1,000 channels · 100M tokens
01 — Purpose

What this system does

Two people operating from different locations need to collaboratively build a list of 1,000 Telegram channel URLs without duplicating effort. This system provides a shared, login-protected web app where both users can submit channel links, see the real-time list, and download it as a backup file.

Phase 2 (automated scraping of 100,000 tokens per channel) is defined as a future feature. The current scope is Phase 1 only: collection, de-duplication, and export.

02 — Users

Who uses the system

U-01There are exactly two users. No registration, no public sign-up. Accounts are created manually at setup time.
U-02Every page and action requires an active login session. Unauthenticated requests redirect to the login screen.
U-03Both users share the same channel list and see each other's additions in real time.

04 — Functional requirements

Channel submission & de-duplication

F-01On submission, the system normalises the URL (strips trailing slashes, lowercases the handle) and checks it against the database before saving.
F-02If the channel already exists, the submission is blocked and a message is shown: "Already in the list — added by [username] on [date]."
F-03If the channel is new, it is saved immediately with the submitting user's name and a timestamp.
F-04The duplicate check is instant and happens before any database write, so two simultaneous submissions of the same link result in only one record.

05 — Export requirements

Download formats

E-01The .txt export contains one channel URL per line, sorted by date added (oldest first). No headers, no metadata.
E-02The .jsonl export contains one JSON object per line. Each object includes: url, added_by, added_at (ISO 8601).
E-03Both exports are snapshots — they reflect the current state of the database at the moment of download. Either user can download at any time.
E-04The filename includes the date and total count, e.g. channels_312_2026-06-22.jsonl.
06 — Future scope

Phase 2 — Scraping (not in scope)

The following is planned but not built in Phase 1. The UI reserves a placeholder section to make the transition clear.

S-01Each collected channel will be scraped for up to 100,000 tokens of text content.
S-02Scraping will use a locking mechanism so both workers can run simultaneously without overlapping channels.
S-03Scrape status per channel (pending, in progress, complete, token count) will be visible in the dashboard.
S-04The .jsonl export from Phase 1 is the intended input format for the Phase 2 scraper.
